## AI 协作契约

**日期：** 2026年4月14日

---

### 第一条：AI 使用原则

- AI 是效率工具，工程师是责任主体
- 所有 AI 生成的输出必须经过人工审查才能合入项目

### 第二条：AI 使用范围

- 允许使用：需求初稿、代码补全、测试用例草案、文档润色、重构建议
- 需额外审查：数据库设计、安全逻辑、权限控制、算法实现、性能优化
- 完全禁止：伪造调研数据、伪造测试结果、抄袭他人作业

### 第三条：过程记录要求

- 所有与 AI 的交互 Prompt 必须保留日志，存放于 `docs/Px/Prompt日志.md`
- Git Commit Message 必须标注：
  - `[AI-assisted]`：AI 生成初稿，人工修改超过 50%
  - `[Human-written]`：完全人工编写
- 每阶段结束前填写《AI 协作反思日志》

### 第四条：代码合入规则

- AI 生成的代码必须通过至少一名非作者团队成员的 Code Review
- 每位成员在每个编码阶段，必须至少有 2 个完全手写的核心函数

### 第五条：违约处理

- 若发现未经审查直接提交 AI 生成代码导致严重 Bug 或逻辑错误，责任人需在团队会议上做 5 分钟复盘分享，讲解该段代码的原理
- 累计 3 次违规者，需请大家喝奶茶一次
