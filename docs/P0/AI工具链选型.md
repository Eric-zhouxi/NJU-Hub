# 2. AI工具链选型

### 2.1 AI编程助手

| 候选工具       | 优点              | 缺点         | 结论 |
| -------------- | ----------------- | ------------ | ---- |
| GitHub Copilot | IDE集成强，补全快 | 付费         | 备用 |
| Cursor         | 多文件理解强      | 学习成本略高 | 采用 |

**选择理由：** 虽然Copilot生态成熟，但团队以编码效率优先，采用Cursor上手快

### 2.2 AI对话工具

| 候选工具 | 优点                     | 缺点             | 结论 |
| -------- | ------------------------ | ---------------- | ---- |
| ChatGPT  | 文档、分析、代码能力均衡 | 需审查输出       | 采用 |
| Claude   | 长文处理好               | 部分场景速度波动 | 备用 |

### 2.3 AI代码审查工具

| 候选工具       | 优点         | 缺点       | 结论 |
| -------------- | ------------ | ---------- | ---- |
| CodeRabbit     | PR审查体验好 | 付费限制   | 采用 |
| Sonar + AI插件 | 静态分析强   | 配置较复杂 | 备用 |

### 2.4 AI测试工具

| 候选工具          | 优点             | 缺点       | 结论   |
| ----------------- | ---------------- | ---------- | ------ |
| ChatGPT           | 测试用例生成灵活 | 需人工筛选 | 采用   |
| Diffblue/同类工具 | 自动单测能力强   | 环境依赖高 | 不采用 |

### 2.5 使用原则

- AI用于提效，不替代判断
- 所有输出必须人工验证
- 核心设计决策由团队负责
