# NJU-Hub "AI 需求盲区挑战" 对比分析表



| **对比维度**            | **AI 生成的需求**                                            | **人工调研发现的需求**                                       | **分析**                                                     |
| ----------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **AI 正确覆盖的需求**   | 1. 核心四大板块：快递代取、二手交易、组队匹配、学习辅导<br> 2. 校园身份认证机制 <br>3. 信用分与双向评价体系 <br>4. 线上私聊/即时沟通功能 | 1. 跑腿代办、二手交易、学习互助、找搭子组队 <br>2. 80%支持学号认证保证安全 <br>3. 建立评价/诚信分机制 <br>4. 压倒性多数（47人）认为需要线上交流 | **模型推理能力体现**：AI 准确捕捉了“校园互助”这一核心场景下最刚需的 C2C 交易、信任背书（校园认证、信用分）以及沟通闭环，这些是所有互助平台的底层基建。 |
| **AI 遗漏的需求**       | 无                                                           | 1. 咨询/问答被选为“最重要”频次最高（12次）<br> 2. 问卷与访谈高频提及“失物招领”<br> 3. 多位受访者明确提出“红黑榜”及“竞赛/志愿信息汇总”<br> 4. 接受“平台积分”作为酬劳（约40人同意） | **为什么 AI 会遗漏？**  Prompt 仅列举了“快递、辅导、二手、组队”，AI 陷入了“过度聚焦”，导致它忽略了校园生活极高频的非交易类需求（问答、失物招领、选课避坑）。同时，AI 缺乏对“学生群体低成本试错”的理解，未能想到“虚拟积分体系”和类似学生自治的“大众评审”。 |
| **AI 捏造/过度的需求**  | 1. 强制且单一的“学号/教务系统”认证 <br>2. 强制的资金平台托管闭环（需预先支付）<br> 3. 全功能的独立 App 设想及高大上的并发指标 | 1. 20%用户反对单一学号认证，担忧隐私，希望能有邮箱/手机等辅助方式 <br>2. 访谈显示用户愿意接受灵活支付（直接面交、打入校园卡、甚至一顿饭/折现） <br>3. 用户更期待轻量级（如完全免费或极低成本）、依托已有生态（如小程序） | **这些需求真的存在吗？** 存在于成熟的商业社会，但在校园水土不服。AI 是用“成熟商业 C2C 平台（如闲鱼/美团）”的逻辑在套用校园场景。实际上，学生间的交易往往更灵活、客单价低，过重的资金托管和强制实名反而会引起反感和降低转化率。 |
| **AI 表述有歧义的需求** | 1. 任务完成的判定标准（AI 设想：接单者点完成，发单者点确认）<br> 2. 纠纷处理机制（AI 设想：申请客服介入，冻结资金） | 1. 实际操作中极易扯皮。调研数据显示，用户强烈要求加入“上传凭证/拍照保存”作为硬性判断标准。<br> 2. 平台定位歧义。AI 预设了有全职客服的商业模式，但问卷显示管理员需由“志愿学生”担任，人工客服介入在此背景下极不现实。 | **歧义在哪里？** AI 描绘的流程在逻辑上是通顺的，但在**“校园落地执行层面”**过于理想化。它忽略了志愿性质的运营团队根本无力承担繁重的“人工客服仲裁”工作，必须依赖产品机制（如拍照强制流转、大众投票）来降低管理成本。 |
