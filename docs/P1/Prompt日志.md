## 2. 核心 Prompt 详情回溯

#### 交互 01：需求种子生成

- **Prompt**: *"请为一个校园互助服务平台生成完整的需求规格说明书，平台支持学生发布和接单各类互助需求（快递代取、学习辅导、二手交易、组队匹配等），目标用户为大学生。请按 IEEE 830 标准格式输出。"*
- **AI 贡献**: 建立了规范的文档结构，明确了基础的交易流（发布-接单-支付）。

#### 交互 03：数据驱动的重构

- **Prompt**: *"基于调研结果和 AI 辅助，编写正式的需求规格说明书。要求标注来源：[AI生成]、[人工补充]、[AI生成+人工修改]。AI 生成后经人工修改的需求，需简要说明修改了什么。"*
- **AI 贡献**: 成功将调研中的“积分制度”、“失物招领”和“学号认证争议”整合进规格说明书，提升了文档的落地价值。

#### 交互 04：深度建模

- **Prompt**: *"绘制系统级用例图（至少覆盖 3 个角色、15 个用例）；为 3 个核心用例编写详细用例描述；编写至少 15 条用户故事，并对每条用户故事进行优先级排序（MoSCoW 方法）。"*
- **AI 贡献**: 将零散的需求转化为可执行的开发逻辑，特别是“拍照留痕”与“仲裁机制”的流程化。

------

## 3. 协作效能评估

- **框架构建速度**: 极高。AI 在分钟级内完成了数千字的 IEEE 标准文档排版。
- **需求深度**: 中等。AI 擅长“通用逻辑”，但在“校园特有场景”（如教师红黑榜、避坑指南）上必须依赖人工注入调研数据。
- **纠错能力**: 高。针对 Mermaid 语法报错，AI 能迅速提供替代方案（如 graph 语法模拟），保证了工程产出的连续性。

------

**日志结语**：

AI 解决了 *“从 0 到 1”* 的结构化问题，而人工通过调研数据的注入解决了 *“从通用到垂直”* 的深度问题。
